# Choi Beta Testing

Цель beta-теста: проверить, что пользователь может пройти полный локальный сценарий Choi без помощи разработчика.

## Перед Тестом

- Используйте два разных аккаунта: покупатель и продавец.
- Для каждого аккаунта подтвердите email.
- После первого входа заполните onboarding: имя, район, обращение.
- Тестируйте на desktop и mobile.

## Сценарий 1: Регистрация И Профиль

1. Откройте `/register`.
2. Создайте аккаунт.
3. Подтвердите email.
4. Войдите в Choi.
5. Выберите район и обращение.
6. Откройте `/profile`.

Ожидаемый результат:
- профиль открывается без ошибки;
- район и имя отображаются;
- статус доверия Choi отображается;
- дата регистрации показывает месяц и год.

## Сценарий 2: Публикация Объявления

1. Откройте `/sell`.
2. Добавьте фото.
3. Заполните обязательные поля.
4. Опубликуйте объявление.
5. Откройте созданное объявление.

Ожидаемый результат:
- при пустых обязательных полях страница прокручивается к ошибке;
- объявление появляется на главной и в профиле продавца;
- фото открываются и листаются.

## Сценарий 3: Поиск И Фильтры

1. Откройте `/search`.
2. Выберите категорию.
3. Выберите район.
4. Используйте фильтры категории.
5. Обновите страницу.

Ожидаемый результат:
- фильтры сохраняются в URL;
- результаты учитывают район и категорию;
- поиск работает по названию, описанию и характеристикам.

## Сценарий 4: Избранное

1. Нажмите сердце на объявлении.
2. Откройте `/favorites`.
3. Уберите объявление из избранного.

Ожидаемый результат:
- счетчик избранного обновляется;
- избранное сохраняется после обновления страницы.

## Сценарий 5: Чат И Встреча

1. Покупатель открывает объявление продавца.
2. Нажимает `Написать`.
3. Отправляет сообщение или быструю фразу.
4. Предлагает время встречи.
5. Продавец принимает время.

Ожидаемый результат:
- чат открывается без пустого состояния вместо загрузки;
- быстрые фразы отправляются сразу;
- мобильное поле ввода не увеличивает страницу;
- после принятия времени видно, что встреча назначена;
- комментарий к встрече отображается.

## Сценарий 6: Сделка И Отзыв

1. Участники договариваются в чате.
2. Продавец отправляет сделку.
3. Покупатель подтверждает или отклоняет.
4. После завершения сделки пользователь оставляет отзыв.

Ожидаемый результат:
- завершенное объявление уходит в архив;
- его нельзя вернуть в активные;
- сделка появляется в истории покупок и продаж;
- отзыв влияет на доверие Choi.

## Сценарий 7: Уведомления

1. Отправьте сообщение с другого аккаунта.
2. Проверьте счетчик сообщений.
3. Откройте `/notifications`.
4. Нажмите `Прочитать всё`.

Ожидаемый результат:
- счетчик сообщений не исчезает из-за прочтения обычных уведомлений;
- уведомления открывают правильный маршрут;
- новые уведомления появляются без перезагрузки, если Realtime включен.

## Сценарий 8: Жалоба И Модерация

1. Откройте чужое объявление.
2. Отправьте жалобу.
3. Администратор открывает `/admin`.
4. Проверьте список жалоб.

Ожидаемый результат:
- обычный пользователь не видит `/admin`;
- moderator/admin видит модерацию;
- `/debug/auth` и `/debug/listings` закрыты для обычных пользователей.

## Что Считать Блокером Beta

- Нельзя войти после подтверждения email.
- Нельзя опубликовать объявление.
- Чужой пользователь не видит опубликованное объявление.
- Чат не открывается между двумя аккаунтами.
- Сделка или бронь ломают статус объявления.
- Обычный пользователь открывает `/admin` или `/debug/*`.
- Production build или GitHub CI падает.
